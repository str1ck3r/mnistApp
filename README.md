# Сайт для распознавания рукописных цифр (MNIST)

Простой сайт на Flask: пользователь рисует цифру на canvas в браузере,
сервер сжимает рисунок до 28×28 и прогоняет через обученную CNN-модель.

## Структура проекта

```
mnist_site/
├── app.py                # Flask-сервер
├── mnist_model.keras     # обученная модель (нужно положить сюда!)
├── templates/
│   └── index.html        # страница с canvas
├── requirements.txt
└── README.md
```

## Шаг 1. Сохрани модель из Colab

В конце своего ноутбука (после обучения, перед закрытием) добавь и выполни:

```python
model.save('mnist_model.keras')

from google.colab import files
files.download('mnist_model.keras')
```

Скачанный файл `mnist_model.keras` положи в папку `mnist_site/` —
рядом с `app.py`.

## Шаг 2. Запуск локально (проверить, что всё работает)

```bash
cd mnist_site
python -m venv venv
source venv/bin/activate        # на Windows: venv\Scripts\activate
pip install -r requirements.txt
python app.py
```

Открой в браузере: http://127.0.0.1:5000

Нарисуй цифру мышкой на холсте и нажми «Распознать».

## Шаг 3. Деплой, чтобы сайт был доступен всем

### Вариант А — Render.com (рекомендую, бесплатно и просто)

1. Создай репозиторий на GitHub и залей туда папку `mnist_site`
   (включая `mnist_model.keras` — если файл больше 100 МБ, см. примечание ниже).
2. Зайди на https://render.com → New → Web Service → подключи свой GitHub-репозиторий.
3. Настройки сборки:
   - **Build Command:** `pip install -r requirements.txt`
   - **Start Command:** `gunicorn app:app`
4. Нажми Deploy. Через несколько минут Render выдаст публичную ссылку
   вида `https://твой-проект.onrender.com` — это и есть ссылка для преподавателя.

### Вариант B — PythonAnywhere (тоже просто, заточен под Flask)

1. Зарегистрируйся на https://www.pythonanywhere.com
2. Загрузи файлы проекта (через Files или Git).
3. В разделе Web → Add a new web app → Flask → укажи путь к `app.py`.
4. Перезапусти веб-приложение — сайт будет доступен на
   `https://твой_логин.pythonanywhere.com`.

### Примечание про размер модели

Твоя модель из реферата компактная (~7 МБ, ~1.2 млн параметров) — это
без проблем загрузится на GitHub и на оба сервиса. Если вдруг будет
больше 100 МБ, GitHub такие файлы не примет напрямую — но для этой модели
такой ситуации не будет.

## Как это работает (для отчёта/защиты)

1. Пользователь рисует на холсте 280×280 пикселей — крупно, чтобы было
   удобно рисовать мышкой или пальцем.
2. При нажатии «Распознать» JavaScript конвертирует рисунок в строку
   base64 (`canvas.toDataURL`) и отправляет на сервер запросом `POST /predict`.
3. На сервере (`app.py`, функция `preprocess_image`):
   - изображение переводится в градации серого;
   - инвертируется (на canvas чёрная линия на белом, а MNIST — белая
     цифра на чёрном фоне);
   - сжимается до 28×28 с антиалиасингом (`Image.LANCZOS`);
   - нормализуется в диапазон [0, 1], как в обучении модели.
4. Подготовленный массив (1, 28, 28, 1) подаётся в `model.predict()`.
5. Сервер возвращает JSON с предсказанной цифрой, уверенностью модели
   и вероятностями по всем 10 классам — это и отображается на странице.
